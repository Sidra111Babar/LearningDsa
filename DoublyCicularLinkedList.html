Doubly Circular Linked List

    A doubly circular linked list is a hybrid of two types of linked lists:
  • Doubly Linked List Each node has two pointers, one to the next node and one to the previous node.
  • Circular Linked List The last node’s next pointer points to the first node, and the first node's previous pointer points to the last node, forming a continuous loop in both directions.
    • In this structure, you can traverse the list in both directions (forward and backward), and no node is designated as the "end" since the list is circular.

Each node contains three components:

1. Data: The value stored in the node.

3. next: Points to the next node in the list. The next pointer of the last node points to the first node instead of nullptr.

3. prev: Points to the previous node in the list. The prev pointer of the first node points to the last node instead of nullptr.

Use Case of Doubly Circular Linked List

    Bidirectional Circular Navigation
  • When you need to continuously cycle through a list in both directions, a doubly circular linked list is effective. For example, in multiplayer games where players take turns in a circle, the direction of turns can change dynamically.
  • Also useful in media players with features like "repeat" and "previous track," allowing seamless navigation in both directions in circular form
  • Circular Buffers: Systems that require both forward and backward cycling of data (such as network buffers or file systems) benefit from the doubly circular structure for efficient navigation.

When Not to Use Doubly CircularLinked List

  1. Memory Overhead Each node in a doubly circular linked list contains two pointers (next and prev), leading to increased memory usage. In applications where memory is a constraint (e.g., embedded systems), a singly circular linked list may be more appropriate because it has only one pointer per node.
  2. Simple Traversal If you only need to traverse in one direction (forward) and do not need circular or bidirectional navigation, a singly linked list or singly circular linked list is more efficient. They are simpler, and you avoid the complexity of managing two pointers per node.
  3. No Circular Navigation Needed If your use case doesn’t require looping back to the beginning or end, a doubly linked list (non-circular) is sufficient. For example, in linear data structures like lists, queues, or stacks, circular behavior is not necessary, and a non-circular list is easier to implement and manage.
  4. Less Complex Operations In scenarios where you mostly perform operations at the head or tail of the list (like in a stack or a simple queue), a singly linked list is often sufficient and easier to implement, without the overhead of managing two pointers (prev and next) for each node.

void DoublyCircularLL::removeDuplicates() {
    if (head == nullptr) {
        cout << "List is empty." << endl;
        return;
    }

    Node *current = head;
    do {
        Node *temp = current->next;
        while (temp != head) {
            if (temp->data == current->data) {
                Node *duplicate = temp;
                temp = temp->next;
                duplicate->prev->next = duplicate->next;
                duplicate->next->prev = duplicate->prev;
                delete duplicate;
            } else {
                temp = temp->next;
            }
        }
        current = current->next;
    } while (current != head);
}

int DoublyCircularLL::count(){
	if (isEmpty()) {
        return 0;
    }

    Node* temp = head;
    int count = 1;
    while (temp != tail){
    	temp = temp->next;
    	count++;
	}

    return count;
}

bool DoublyCircularLL::isEmpty(){
	return (head == NULL);
}
void DoublyCircularLL::insertAtTail(int value){
	if (DoublyCircularLL::isEmpty()) {
        Node *newnode = new Node(value);
        head = newnode;
        head->next = head;
        head->prev = head;
        tail = head;
        return;
    }
    Node *newnode = new Node(value);
    newnode->prev = tail;
    tail->next = newnode;
    newnode->next = head;
    head->prev = newnode;
    tail = newnode;
}
void DoublyCircularLL::insertAtHead(int value) {
    if (DoublyCircularLL::isEmpty()) {
        Node *newnode = new Node(value);
        head = newnode;
        head->next = head;
        head->prev = head;
        tail = head;
    } else {
    	Node *newnode = new Node(value);
        Node* lastNode = head->prev;         
            newnode->next = head;            
            newnode->prev = lastNode;        
            lastNode->next = newnode;         
            head->prev = newnode;             
            head = newnode;
    }
}
