from django.shortcuts import render
from .forms import StudentForm
from .models import Student2

def index(request):
    if request.method == "POST":
        data = StudentForm(request.POST)  # Capturing form data
        if data.is_valid():  # Checking if data is valid
            print(data.cleaned_data)  # Printing the cleaned data
            name = data.cleaned_data['name']
            age = data.cleaned_data['age']
            
            # Storing data in the database
            Student2.objects.create(name=name, age=age)
    else:
        print(request.method)  # Print the request method if it's not POST
    
    form = StudentForm()  # Creating a blank form
    context = {'form': form}
    return render(request, 'index.html', context)

Explanation

    The function index(request):
  • Checks if the request method is "POST".
  • Retrieves form data using StudentForm(request.POST).
  • Validates the data using .is_valid().
  • Extracts name and age using .cleaned_data.
  • Saves the data into the Student2 database table.
  • If the request is not POST, it prints the request method (GET).
  • Creates an empty form (StudentForm) and passes it to index.html.

Forms directly from Models

In Django, forms can be created directly from models using ModelForm. This allows the form to automatically generate fields based on the model, reducing redundancy.By using this instead of creating a form in forms.py you can directly create a form from a model.

class StudentForm(forms.ModelForm):
    class Meta:
        model = Student2  # Connects the form to the Student model
        fields = '__all__'  # Includes all fields from the model

from django.shortcuts import render
from .forms import StudentForm  # Import the form

def index(request):
    form = StudentForm()  # Create an empty form
    if request.method == 'POST':  # Check if form is submitted
        data = StudentForm(request.POST)  # Bind data to the form
        if data.is_valid():  # Validate the form
            data.save()  # Save data to the database
    return render(request, 'index.html', {'form': form})  # Pass form to template

How the Data Flows

  1. The user fills out the form on index.html and clicks "Submit."
  2. The browser sends a POST request to the index view in views.py.
  3. The view processes the form data
    • Validates the data.
    • Stores the data in the Student2 database table.
    • Renders index.html with an empty form.
  4. If the user visits the page without submitting, the form is displayed empty.

Security in Django Forms

    CSRF Protection: Django requires {% csrf_token %} to prevent malicious attacks. Form Validation: .is_valid() ensures only valid data is stored in the database.

Request Flow:

  • Frontend to Backend → The form submits data using POST.
  • Backend to Frontend → The server returns an HTML page using GET.
