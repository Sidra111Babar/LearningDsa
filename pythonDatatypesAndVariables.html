Vairables and Datatypes

Variables

A variable is a container for storing data values. Python does not require you to declare variables with specific data types, making it dynamically typed.

    Naming Rules:
  • Must start with a letter (a-z, A-Z) or an underscore _.
  • Can contain letters, numbers, and underscores but cannot start with a number.
  • Case-sensitive (MyVar and myvar are different).

Dynamic Typing: Variables can change types dynamically.


    x = 10  # Integer
    x = "Hello"  # Now a String

Example


    name = "Alice"  # String
    age = 25        # Integer
    height = 5.7    # Float
    is_student = True  # Boolean

Datatypes

Datatypes are of two types builtin and user defined

Built in

In Python, datatypes define the type of data a variable can hold and the operations that can be performed on that data.

Text Type

  • str: Used for storing and manipulating textual data.

Numeric Types

  • int: Represents whole numbers.
  • float: Represents decimal or floating-point numbers.
  • complex: Represents complex numbers with real and imaginary parts.

Sequence Types

  • list: An ordered, mutable collection of items.
  • tuple: An ordered, immutable collection of items.
  • range: Represents a sequence of numbers, often used in loops.

Mapping Type

  • dict: A collection of key-value pairs for efficient data retrieval.

Set Types

  • set: An unordered collection of unique elements.
  • frozenset: Similar to a set, but immutable and hashable.

Boolean Type

  • bool: Represents Boolean values, True or False.

Binary Types

  • bytes: Represents an immutable sequence of bytes, often used for binary data.
  • bytearray: Similar to bytes but mutable.
  • memoryview: Provides a memory-efficient view of binary data without copying it.

None Type

  • NoneType: Represents the absence of a value.

  • Set In Python, a set is a collection of unordered, unique, and mutable elements. Sets are useful when you need to store items without duplicates and perform mathematical operations like union, intersection, and difference.
      Characteristics
    • Unordered:The items in a set are not stored in any particular order.
    • Unique:Duplicate elements are not allowed.
    • Mutable:You can add or remove elements from a set.
    • Immutable elements:The items in a set must be immutable (e.g., numbers, strings, tuples), but the set itself is mutable.
    
    # Creating a set with unique elements
    my_set = {1, 2, 3, 4, 5}
    print(my_set)  # Output: {1, 2, 3, 4, 5}
    
    # Creating a set with duplicates (duplicates are automatically removed)
    my_set = {1, 2, 2, 3, 4}
    print(my_set)  # Output: {1, 2, 3, 4}
    
    # Creating an empty set (use set(), not {})
    empty_set = set()
    print(empty_set)  # Output: set()                                    
                                
    
    #Union (|) of Sets
    set1 = {1, 2, 3}
    set2 = {3, 4, 5}
    union_set = set1 | set2  # OR set1.union(set2)
    print(union_set)  # Output: {1, 2, 3, 4, 5}
                                
    
    intersection_set = set1 & set2  # OR set1.intersection(set2)
    print(intersection_set)  # Output: {3}
                                
    
    # Finds elements in one set but not in the other.                               
     difference_set = set1 - set2  # OR set1.difference(set2)
     print(difference_set)  # Output: {1, 2}
                                
    
    # Finds elements not common to both sets.(Symmmetric Difference)    
    symmetric_difference_set = set1 ^ set2  # OR set1.symmetric_difference(set2)
    print(symmetric_difference_set)  # Output: {1, 2, 4, 5}                                    
                                
    
    # Adding and Removing elements
    my_set = {1, 2, 3}
    my_set.add(4)
    print(my_set)  # Output: {1, 2, 3, 4}
    # remove() raises an error if the element doesn't exist.
    # discard() doesn't raise an error if the element doesn't exist.    
    my_set.remove(3)   # Removes 3
    my_set.discard(5)  # No error even if 5 is not in the set
                                
    
    my_set.clear()
    print(my_set)  # Output: set()
    Method Description
    add() Adds an element to the set.
    remove() Removes a specified element (raises error if not found).
    discard() Removes a specified element (no error if not found).
    union() Returns the union of sets.
    intersection() Returns the intersection of sets.
    difference() Returns the difference between sets.
    symmetric_difference() Returns elements not common to both sets.
    clear() Removes all elements from the set.
    pop() Removes and returns an arbitrary element.
  • dict In Python, a dictionary is a versatile collection that stores data in the form of key-value pairs, making it ideal for organizing information in a structured way. Each key serves as a unique identifier for its associated value, ensuring efficient and fast data retrieval. Keys in a dictionary must be immutable, meaning they can only be of types like strings, numbers, or tuples, while values are mutable and can belong to any data type, allowing for flexibility in data storage. Although dictionaries were unordered prior to Python 3.7, they now maintain the order of key-value pairs as they are added. This combination of structured organization, fast access, and flexibility makes dictionaries a powerful tool for managing data in Python.

    Operations: Append,Remove and Sort

    
    # Creating a dictionary
    my_dict = {
        "name": "Alice",
        "age": 25,
        "city": "New York"
    }
    print(my_dict)  # Output: {'name': 'Alice', 'age': 25, 'city': 'New York'}
                                
    
    # Two Access Methods
    print(my_dict["name"])  # Output: Alice
    print(my_dict["age"])   # Output: 25
    # Safe way to access a value (returns None if key does not exist)
    print(my_dict.get("city"))  # Output: New York
    print(my_dict.get("country", "Not Found"))  # Output: Not Found
                                
    
    # Adding or Updating key-value pair
    my_dict["country"] = "USA"  # Add new key-value pair
    my_dict["age"] = 26         # Update existing key
    print(my_dict)  # Output: {'name': 'Alice', 'age': 26, 'city': 'New York', 'country': 'USA'}
                                
    
    # Removing key-value pair
    removed_value = my_dict.pop("city")
    print(removed_value)  # Output: New York
    
    # Using del (does not return value)
    del my_dict["age"]
    
    # Using popitem() (removes the last inserted item in Python 3.7+)
    last_item = my_dict.popitem()
    print(last_item)  # Output: ('country', 'USA')                                    
                                
    
    my_dict.clear()
    print(my_dict)  # Output: {}
    • Method Description
    get(key[, default]) Returns the value for the key, or a default if not found.
    keys() Returns a view object of all keys.
    values() Returns a view object of all values.
    items() Returns a view object of key-value pairs.
    pop(key[, default]) Removes the key and returns its value.
    update(other_dict) Updates the dictionary with key-value pairs from another dictionary or iterable.
    setdefault(key[, default]) Returns the value of the key if it exists, else inserts it with the default value.
    copy() Returns a shallow copy of the dictionary.

User Defined

In Python, user-defined data types are created by the programmer to represent complex data structures that are not provided by the built-in types. The primary tools for creating user-defined data types in Python are classes. By using classes, you can define custom behaviors and attributes for your data structures.

  def __init__(self, name, age):
  self.name = name  # Attribute for name
  self.age = age    # Attribute for age
  def greet(self):
  print(f"Hello, my name is {self.name} and I am {self.age} years old.")
  # Create an instance of Person
  p = Person("Alice", 25)
  p.greet()
