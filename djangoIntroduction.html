Virtual Environment

A virtual environment in Python is an isolated environment that allows you to install and manage dependencies separately from the system-wide Python installation. This helps avoid conflicts between different projects that require different package versions.

Why to use Virtual Environment

  • Dependency Management: Keeps project dependencies separate.
  • Avoid Conflicts: Different projects can have different versions of the same library.
  • Easier Deployment: Ensures consistency across development and production environments.

Steps to Create Virtual Environment and start django app

  • Create Create Virtual Environment by giving command python -m venv venv
    • 
python -m venv venv
  • Activate Activate the Virtual Environment by giving command .\venv\Scripts\activate
    • 
.\venv\Scripts\activate
  • Install Install library by giving command pip install library name
    • 
pip install django
  • Start Django Project Start by giving command django-admin startproject firstproject
    • 
django-admin startproject firstproject
  • Start Django appStart the app by giving command python manage.py startapp home
    • 
python manage.py startapp home
  • Run Server Run by giving command python manage.py runserver
    • 
python manage.py runserve

Explanation of Files in home folder in Django

  1. migrations/ Folder

    This folder contains migration files, which are used for database schema changes.

    • Migrations help Django track changes to the models (database structure) and apply them to the database without writing raw SQL queries.
    • Django automatically creates migration files when you run: 
      
python manage.py makemigrations
    • To apply migrations and update the database, use: 
      
python manage.py migrate
    • Each migration file is named like 0001_initial.py, 0002_auto.py, etc.
  2. __init__.py
    • This is a special Python file that tells Python to treat the directory as a package.
    • In Django apps, __init__.py allows the app to be recognized as a module.
    • Without this file, Python wouldn't treat the folder as an importable module.
    • This file is mostly empty but necessary for Django to work properly.
  3. admin.py
    • Django provides a built-in Admin Panel, which allows managing database records through a web interface.
    • admin.py is used to register models so they appear in the Django admin panel.
    • 
 from django.contrib import admin
 from .models import Product  # Import the model  
 admin.site.register(Product)  # Register the model
    • After registering, the Product model will be visible in the Django Admin Panel.
  4. apps.py
    • This file contains the configuration for the app.
    • Every Django app has an apps.py file, which tells Django that the app exists.
      • 
from django.apps import AppConfig

class HomeConfig(AppConfig):
    default_auto_field = 'django.db.models.BigAutoField'
    name = 'home'
    • If you create another app (e.g., accounts), Django will generate accounts/apps.py with AccountsConfig.
  5. models.py
    • This file is where you define the database structure using Django's ORM (Object-Relational Mapping).
    • Instead of writing raw SQL queries, you define models (tables), and Django handles database interactions.
    • 
from django.db import models

class Product(models.Model):
    name = models.CharField(max_length=100)
    price = models.DecimalField(max_digits=10, decimal_places=2)
    created_at = models.DateTimeField(auto_now_add=True)
    • Running python manage.py makemigrations will create a migration file for this model.
    • Running python manage.py migrate will apply the model to the database.
  6. tests.py
    • This file is used to write unit tests for the app.
    • Unit tests help ensure that functions, views, and APIs work correctly.
    • 
from django.test import TestCase
from .models import Product

class ProductTestCase(TestCase):
    def test_product_creation(self):
        product = Product.objects.create(name="Laptop", price=1000)
        self.assertEqual(product.name, "Laptop")
    • Running python manage.py test will execute all test cases.
  7. views.py
    • This file contains the main logic of the application.
    • It handles requests and returns responses.
    • 
from django.http import HttpResponse

def home(request):
    return HttpResponse("Welcome to the Home Page!")
    • This function is a view that returns a simple text response.
    • In a real application, views usually return HTML templates or JSON responses.
  8. urls.py(Not Automatically Created, But Important)
    • This file defines URL patterns for the app.
    • You need to create this file manually inside the app directory.
      • 
from django.urls import path
from .views import home

urlpatterns = [
    path('', home, name='home'),
]
    • This maps the root URL (/) to the home view.
  9. Registering the App in settings.py
    • Every new app must be added to INSTALLED_APPS in settings.py.
      • 
  INSTALLED_APPS = [
      'django.contrib.admin',
      'django.contrib.auth',
      'home',  # Register the 'home' app here
  ]
