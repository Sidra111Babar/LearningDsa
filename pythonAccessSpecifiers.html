Access Specifiers

In Python, access specifiers are used to control the accessibility of class members (attributes and methods). Python doesn't have explicit access specifiers like public, protected, or private as in languages like Java or C++, but it achieves similar functionality through naming conventions and special syntax.

  • Public

    These members are accessible from anywhere (inside and outside the class).All class members are public by default

    Not Copied!

    
class Person:
    def __init__(self, name, id):
        self.Name = name  # Public member
        self.ID = id   # Public member

    def display(self):  # Public method
        print(f"Name: {self.Name}, ID: {self.ID}")

# Accessing public members
obj = Person("Sidra", 1)
print("Name:", obj.Name)  # Access public attribute
obj.display()              # Call public method

  • Protected

    Protected members are indicated by a single underscore _ as a prefix.Accessible within the class and its subclasses. The _ prefix is just a convention and does not enforce true restriction.

    Copied!

    Not Copied!

    
class Person:
    def __init__(self, name, age):
        self._name = name      # Protected member
        self._age = age      # Protected member

    def _display(self):   # Protected method
        print(f"Name: {self._name}, Age: {self._age}")

class Student(Person):
    def displayDetails(self):
        print(f"Name: {self._name}, Age: {self._age}")  # Access protected attribute
        self._display()         # Call protected method

# Accessing protected members
obj = Student("Anila",15)
obj.displayDetails()

  • Private

    Private members are indicated by a double underscore __ as a prefix. Accessible only within the class. Python achieves this by name mangling, which renames the attribute to include the class name as a prefix.

    Not Copied!

    
class Person:

# Private members
__name = None
__age = None

# Constructor
def __init__(self, name, age):
    self.__name = name
    self.__age = age

# Private member function
def __displayDetails(self):
    # Accessing private data members
    print(f"Name: {self.__name}, Age: {self.__age}")
# Public member function to access private function
def accessPrivateFunction(self):
    # Accessing private member function
    self.__displayDetails()

# Creating object
person_obj = Person("Anila", 15)

print(dir(person_obj))
print("")

# Attempting to access private members directly (will throw an error). All these are private.
# person_obj.__name
# person_obj.__age
# person_obj.__displayDetails()

# Accessing private members using name mangling
print(person_obj._Person__name)
print(person_obj._Person__age)
person_obj._Person__displayDetails()

print("")

# Accessing private method through a public member function
person_obj.accessPrivateFunction()
