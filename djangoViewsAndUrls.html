Views and Urls in Django

In Django, views handle the logic of what gets displayed to the user, while URLs map specific paths to those views. Views are defined in views.py using functions or class-based approaches. A simple view returns an HttpResponse or renders an HTML template. URLs are configured in urls.py, where path() or re_path() functions link URLs to views. For modularity, each Django app has its own urls.py, which is then included in the project's main urls.py using include(). This separation allows Django to efficiently manage web requests and responses

Dynamic Routing

Dynamic routing in Django allows URLs to accept parameters, making them more flexible. This is done using path converters in urls.py.

Views.py


def dynamic_route(request, number):
    result = ""
    
    # Generate multiplication table
    for i in range(1, 11):  
        result += f"{number} * {i} = {number * i}
"  

    count = {}  
    number_str = str(number)  # Convert number to string for iteration

    for n in number_str:  
        count[n] = count.get(n, 0) + 1  

    result += f"
Digit Count: {count}"  
    return HttpResponse(result)

Urls.py


from django.urls import path
from .views import dynamic_route

urlpatterns = [
    path('dynamic_route//', dynamic_route, name='dynamic_route'),
]

Note: visit http://127.0.0.1:8000/dynamic_route/5/

Output

Conclusion

This view takes number as a parameter, generates its multiplication table, counts the occurrences of each digit, and returns the results as an HTTP response. For example, accessing /dynamic_route/5/ will output the table for 5 and digit count {‘5’: 1}. This approach helps create dynamic, data-driven web pages based on user input.
