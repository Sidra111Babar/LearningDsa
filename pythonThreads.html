Threads

Threads are lightweight units of execution within a program that allow you to perform multiple tasks at the same time.

  • A program can have one or more threads running independently.
  • Each thread can handle a separate task, allowing your program to do many things simultaneously.

Example of Single Thread

    Imagine you're cooking a meal:
  • You start boiling water.
  • While waiting for it to boil, you chop vegetables.
  • After the water boils, you cook pasta.

If you're the only person cooking (a single thread), you have to finish one task before moving to the next. If chopping takes 5 minutes and boiling takes 10 minutes, you'll spend 15 minutes in total.

Example of Multi-threading

    Now imagine you have a helper:
  • You boil water.
  • Your helper chops vegetables at the same time.

Both tasks happen simultaneously, reducing the total time spent. This is multi-threading.

Why Use Threads

    Threads are useful because they:
  • Improve Speed: Tasks that can run independently, such as downloading files or processing data, can run simultaneously, reducing wait times.
  • Make Programs Responsive: For example, a thread can keep a user interface responsive while another thread processes data in the background.
  • Efficient Resource Use: Threads share the same memory space, so creating and managing threads is faster and requires fewer resources than running separate programs.

Explanation

  • A program starts with a main thread (the default thread that runs your program).
  • You can create additional threads to perform tasks in parallel.
  • Threads share the same memory and resources of the program, so they're lightweight compared to processes.

import time  # Importing time to measure execution time and simulate delays
import threading  # Importing threading to create and manage threads

# Define the function that simulates a task taking 1 second
def doSomething():
    print(f"Thread name is {threading.current_thread().name} with id - {threading.get_ident()}")
    print("Doing something")  # Indicate task start
    time.sleep(1)  # Simulate a task that takes 1 second
    print("Done")  # Indicate task completion

# Without using threads
print("Executing without threads...")
start_no_thread = time.perf_counter()  # Start time for non-threaded execution

doSomething()  # Call the function
doSomething()  # Call the function again

finish_no_thread = time.perf_counter()  # End time for non-threaded execution
print(f"\nFinished without threads in {round(finish_no_thread - start_no_thread, 2)} seconds(s)\n")

# Using threads
print("Executing with threads...")
start_with_thread = time.perf_counter()  # Start time for threaded execution

# Create two threads to execute the doSomething function
t1 = threading.Thread(target=doSomething)
t2 = threading.Thread(target=doSomething)

t1.start()  # Start the first thread
t2.start()  # Start the second thread

t1.join()  # Wait for the first thread to finish
t2.join()  # Wait for the second thread to finish

finish_with_thread = time.perf_counter()  # End time for threaded execution
print(f"\nFinished with threads in {round(finish_with_thread - start_with_thread, 2)} seconds(s)")

Explanation

  • Without Thread
    • 
Do something -> 1s  
Do something -> 1s  
Total: 2s
  • With Thread
    • 
Do something (Thread-1) + Do something (Thread-2) -> 1s  
Total: 1s
