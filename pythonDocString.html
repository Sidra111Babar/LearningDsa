Docstring

A docstring in Python is a special kind of comment used to describe what a function, class, or module does. It is enclosed within triple quotes (""" """ or ''' ''') and is placed as the first statement inside the function, class, or module.

Docstrings serve as documentation and can be accessed at runtime using the __doc__ attribute.

Why Use

  • Documentation: Describes the purpose and functionality of modules, classes, and functions directly in the code.
  • Improved Readability: Makes the code self-explanatory, reducing the need for external documentation.
  • Ease of Maintenance: Helps in debugging, extending, or modifying the code in the future.
  • Automated Documentation: Enables tools like Sphinx and pydoc to generate detailed documentation automatically.
  • Runtime Accessibility: Allows retrieving documentation dynamically using the __doc__ attribute.
  • Encourages Good Practices: Promotes clear thinking and better design during development.
  • Simplifies Code Review: Assists reviewers or other developers in understanding the intent and usage of code components.

def add_numbers(a, b):
    """
    Adds two numbers together.

    Parameters:
        a (int): The first number.
        b (int): The second number.

    Returns:
        int: The sum of the two numbers.
    """
    return a + b
print(add_numbers.__doc__)

class Calculator:
    """
    A simple calculator class for basic arithmetic operations.

    Methods:
        add(a, b): Returns the sum of a and b.
        subtract(a, b): Returns the difference of a and b.
    """

    def add(self, a, b):
        """Adds two numbers."""
        return a + b

    def subtract(self, a, b):
        """Subtracts two numbers."""
        return a - b
print(Calculator.__doc__)
