Match Case

Case matching in Python refers to the use of the match statement, which was introduced in Python 3.10. It allows pattern matching, a powerful way to compare values against patterns and execute code based on which pattern matches. It's similar to switch-case in other programming languages but more expressive and versatile.

Syntax

Copied!

Not Copied!


match variable:
    case pattern1:
        # Action for pattern1
    case pattern2:
        # Action for pattern2
    case _:
        # Default action (if no patterns match)

Explanation

  • Patterns: Patterns can be literals, variable bindings, wildcards, or even data structures.
  • Wildcard_: Acts as a default case if no other patterns match.
  • Destructuring: You can extract values from sequences or objects.

Benefits

  • Simplifies complex conditional logic.
  • Improves code readability with structured pattern matching.
  • Handles data destructuring elegantly.

    • Limitations

    • Available only in Python 3.10 and newer.
    • Overuse can make simple logic overly complex.

    Example 1

    Not Copied!

    
fruit = "apple"

match fruit:
    case "apple":
        print("This is an apple!")
    case "banana":
        print("This is a banana!")
    case _:
        print("Unknown fruit")

    Real World Banking Example

    We'll design a simple banking system where the user's request determines which function to call.

# Functions for each operation
def deposit(amount):
    print(f"Depositing ${amount} to your account.")
    # Logic for deposit

def withdraw(amount):
    print(f"Withdrawing ${amount} from your account.")
    # Logic for withdrawal

def check_balance():
    print("Checking account balance...")
    # Logic for checking balance

def unknown_request():
    print("Unknown request. Please try again.")

# Banking operation handler
def handle_request(request):
    match request:
        case {"type": "deposit", "amount": amount} if amount > 0:
            deposit(amount)
        case {"type": "withdraw", "amount": amount} if amount > 0:
            withdraw(amount)
        case {"type": "check_balance"}:
            check_balance()
        case _:
            unknown_request()

# Example requests
requests = [
    {"type": "deposit", "amount": 100},
    {"type": "withdraw", "amount": 50},
    {"type": "check_balance"},
    {"type": "withdraw", "amount": -10},  # Invalid
    {"type": "transfer"}  # Unsupported
]

# Process each request
for req in requests:
    handle_request(req)

    Explanation

    The above implementation demonstrates a banking system that processes user requests using Python's match statement. It supports operations like deposit, withdrawal, and balance checks, each tied to a specific function. The match statement simplifies the logic by matching requests to predefined patterns, such as {"type": "deposit", "amount": amount} for deposits and ensures conditions like amount > 0 are met using guards. If a request doesnâ€™t match any known pattern, a default case handles invalid or unsupported operations. This approach ensures clean, readable, and modular code while effectively validating and routing user requests to appropriate functions.

    Class Base Example

    You can match specific attributes of a class using pattern matching and guards.

    Copied!

    Not Copied!

    
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

person = Person("Alice", 25)

match person:
    case Person(name="Alice", age=age) if age >= 18:
        print(f"{person.name} is an adult")
    case Person(name, age):
        print(f"{name} is underage")
    case _:
        print("Unknown person")
