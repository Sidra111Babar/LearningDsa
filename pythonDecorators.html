Decorators

Decorators in Python are a design pattern that allows you to modify or extend the behavior of a function or method without permanently modifying its code. Decorators are typically used to wrap another function, adding some functionality before or after the function is executed. In Python, decorators are applied using the @decorator_name syntax.

# A simple decorator that prints a message before and after a function call:
def my_decorator(func):    # here func is say_hello() which is defined below
    def wrapper():
        print("Before the function is called.")
        func()             # calling of say_hello
        print("After the function is called.")
    return wrapper
# Decorators takes another function as argument here decorator is taking say_hello as argument.
@my_decorator
def say_hello():
    print("Hello, world!")
say_hello()
# Before the function is called.
# Hello, world!
# After the function is called.

Access Control: Restrict access based on conditions

# example when wrapper takes an Argument
def require_admin(func):
    def wrapper(user):
        if user != "admin":
            print("Access denied!")
            return
        return func(user)
    return wrapper

@require_admin
def view_dashboard(user):
    print(f"Welcome, {user}!")

view_dashboard("guest")  # Access denied!
view_dashboard("admin")  # Welcome, admin!

Type Error Handling: to prevent the type error.

This code defines a decorator, int_only, which ensures that the argument passed to the decorated function is of type int. When the square function is called, the decorator's wrapper function first checks the type of the argument using the built-in function isinstance(). If the argument is not an integer, it raises a TypeError using the built-in function raise, along with an error message that includes the invalid type, retrieved using type() and .__name__. If the type check passes, the original function (square) is called, and it calculates the square of the number by returning n * n. The @int_only decorator effectively enforces type safety for the square function without modifying its core logic.

    Role of Builtin Functions

  • isinstance(arg, int): Checks if the argument (arg) is an instance of the specified type (int).Ensures type safety by validating the argument before calling the original function.
  • type(arg): Returns the type of the object (arg), such as or .Used here to identify the actual type of the argument if it fails the type check.
  • .__name__: An attribute of the type object returned by type().Extracts the type name (e.g., "int", "str") as a string for user-friendly error messages.
  • raise: Raises an exception (in this case, TypeError) when the type check fails. Stops further execution and notifies the user of the invalid argument type.
  • print(): Outputs the result

Not Copied!


def int_only(func):
    def wrapper(arg):
        if not isinstance(arg, int):  # Check if the argument is an integer
            raise TypeError(f"Argument must be of type int, got {type(arg).__name__}")
        return func(arg)
    return wrapper

@int_only
def square(n):
    return n * n

# Test cases
print(square(4))    # Works: Outputs 16
print(square("4"))

Execution time of a function: Measure the time a function takes to execute

Explanation

The timing_decorator wraps the original function, measuring the time before and after its execution to calculate and print the time taken. It takes any number of positional and keyword arguments (*args and **kwargs), passing them to the original function to maintain its functionality.

Builtin Functions Used

  • time.time(): Returns the current time in seconds since the Epoch (typically January 1, 1970). It's used here to capture the start and end times for measuring execution duration.
  • print(): Output the result

Use of *args and **kwargs:

*args: Allows the function to accept an arbitrary number of positional arguments (passed as a tuple). It makes the function flexible, so it can handle different numbers of arguments without explicitly defining them.**kwargs: Allows the function to accept an arbitrary number of keyword arguments (passed as a dictionary). It enables the function to be extended in the future without breaking its signature.

Advantages of Using *args and **kwargs

Flexibility: Functions can accept any number of arguments, making them adaptable to different scenarios.

Reusability: The function can work with different sets of arguments without modification.

Maintainability: They allow future extensions without changing the function's structure or breaking the existing code.

import time
def timing_decorator(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} with args {args} and kwargs {kwargs} took {end_time - start_time:.4f} seconds")
        return result
    return wrapper
@timing_decorator
def slow_function():
    time.sleep(2)  # Simulate a slow task
    print("Finished slow function!")
slow_function()

@timing_decorator
def add_numbers(a, b, c):
    time.sleep(1)  # Simulate a slow task
    total = a + b + c
    print(f"Sum of {a}, {b}, and {c}: {total}")
    return total

# Call the decorated function
result = add_numbers(5, 10, 15)
print("Result:", result)
