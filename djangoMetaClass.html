Meta Classes in Django

Meta classes in Django are used to define model-specific behaviors such as database table names, ordering, constraints, permissions, and more. These allow us to control how a model behaves at the database and Django admin levels without modifying the database schema explicitly.

db_table

By default, Django creates database table names in the format appname_modelname (e.g., home_book). However, using db_table, we can specify a custom table name.

    author = models.CharField(max_length=255)

    class Meta:
        db_table = 'book'  # This will rename the table from 'home_book' to 'book'

index_together

The index_together option in Django's Meta class is used to define composite indexes on multiple columns in a database. It helps improve the efficiency of queries that filter or order results based on multiple fields.

abstract

Abstract models define common fields that multiple models can inherit, without creating a table for the base model.

Copied!

Not Copied!


class Human(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
    phone_number = models.CharField(max_length=100)
    image = models.CharField(max_length=100)

    class Meta:
        abstract = True  # No table will be created for Human

class Engineer(Human):
    technology = models.CharField(max_length=100)

class HR(Human):
    responsibility = models.CharField(max_length=100)

class Investor(Human):
    money = models.CharField(max_length=100)

constraints

Adds advanced validation to prevent invalid data entry.

Not Copied!


from django.db.models import CheckConstraint, Q

class Student(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
    gender = models.CharField(max_length=10)

    class Meta:
        constraints = [
            CheckConstraint(check=Q(age__gte=18), name='age_gte_18')
        ]

proxy

Proxy models in Django allow you to change the behavior of an existing model without modifying its fields or database schema. Essentially, a proxy model is a subclass of another model, but it does not create a separate database table. Instead, it operates on the same table while allowing customization, such as modifying the default manager, changing ordering, or adding custom methods.

class CollegeStudent(Student):
    class Meta:
        proxy = True  # No new table will be created

        constraints = [
            CheckConstraint(check=Q(age__gte=18), name='age_gte_18')
        ]
