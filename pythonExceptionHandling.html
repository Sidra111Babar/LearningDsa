Exception Handling in Python

Exception handling in Python is a way to manage errors that occur during program execution. The try-except block is used to catch and handle exceptions, allowing the program to continue running instead of crashing.

Syntax

  • try: The block where you write code that may cause an exception.
  • except: The block that handles the exception. You can specify the type of exception to handle (e.g., ValueError, TypeError) or handle all exceptions by using a generic except.
  • finally: The block that is executed regardless of whether an exception occurred or not.
  • else: The block that is executed if no exceptions occur in the try block.
  • raise: The keyword used to raise an exception explicitly.

Example 3

try with else and finally

finally: The finally keyword in Python is used in a try-except block to define a block of code that will always execute, regardless of whether an exception was raised or not. This is particularly useful for cleanup operations, such as closing files, releasing resources, or other tasks that must be performed regardless of the outcome of the try block.

Not Copied!


try:
    num = int(input("Enter an integer: "))
    result = 10 / num
except ValueError:
    print("Invalid input! Please enter a valid integer.")
except ZeroDivisionError:
    print("Division by zero is not allowed.")
else:
    print(f"Result: {result}")  # Runs only if no exceptions occur
finally:
    print("Execution complete.")  # Always runs

Real World Exmaple

Raise keyword: In Python, the raise keyword is used to explicitly trigger an exception. This allows you to manually raise an error when a specific condition is met, instead of relying solely on built-in errors or exceptions that occur naturally in the code. You can raise both built-in exceptions and custom exceptions.

This code simulates a payment processing system with custom exception handling. It defines a custom exception PaymentGatewayError for payment gateway-related errors. The process_payment function checks if the entered amount is valid, simulates potential network failures, and randomly triggers errors such as ValueError for invalid amounts, ConnectionError for network issues, and PaymentGatewayError for payment gateway failures. The function handles these exceptions with specific error messages and includes a finally block to indicate the completion of the transaction attempt. The user is prompted to enter a payment amount, and the function simulates a payment process with potential failures.

def process_payment(amount):
    try:
        # Simulate payment processing
        if amount <= 0:
            raise ValueError("Amount must be greater than zero.")
        print(f"Processing payment of ${amount}...")
        
        # Simulate random network failure
        import random
        if random.choice([True, False]):
            raise ConnectionError("Network issue during payment.")
        
        # Simulate payment gateway error
        if random.choice([True, False]):
            raise PaymentGatewayError("Payment gateway is down.")
        
        print("Payment processed successfully!")
    except ValueError as ve:
        print(f"Invalid payment amount: {ve}")
    except ConnectionError as ce:
        print(f"Payment failed due to network error: {ce}")
    except PaymentGatewayError as pge:
        print(f"Payment gateway error: {pge}")
    finally:
        print("Transaction attempt complete. Please try again if necessary.")

# Simulate a payment attempt
amount = float(input("Enter payment amount: "))
process_payment(amount)
