Polymorphism

The term polymorphism comes from the Greek words poly (many) and morphe (forms). In programming, polymorphism refers to the ability of a single function, method, or operator to behave differently depending on the type of object or data it is working with. In Python, it allows you to use the same name (for a method, function, or operator) to perform different actions based on the object or data type.

Example

  • A single method name like speak() can be used by different classes (e.g., Dog and Cat), but each class defines its own version of the method.
  • The same operator + can add numbers or concatenate strings. It depends on the data types involved.

Types of Polymorphism

  • Polymorphism with Built-in Functions:

    Polymorphism allows built-in Python functions to work seamlessly with different types of objects, as long as those objects support the required operations. One common example of this is the len() function, which is inherently polymorphic.

    The len() function is used to return the length (number of items) of an object. However, the behavior of len() is polymorphic because it can operate on different types of objects, such as:

    • Strings
    • Lists
    • Tuples
    • Dictionaries
    • Custom objects (if they implement the __len__() method)
    
#  Using len() with Strings:                          
text = "Hello"
print(len(text))  # Output: 5
                            
    
# Using len() with Lists:
numbers = [10, 20, 30, 40]
print(len(numbers))  # Output: 4                               
                            
    
# Using len() with Tuples:
items = (1, 2, 3, 4, 5)
print(len(items))  # Output: 5                                                         
                            
    
# Using len() with Dictionaries:
dictionary = {"key1": "value1", "key2": "value2"}
print(len(dictionary))  # Output: 2                                                      
                            
    
# Using len() with Custom Objects:
# To make len() work with custom objects, you can implement the __len__() method in your class. Python's polymorphism allows the len() function to interact with the custom object by calling its __len__() method.
class MyCollection:
    def __init__(self, items):
        self.items = items
    
    def __len__(self):
        return len(self.items)

# Custom Object
collection = MyCollection([1, 2, 3, 4, 5])
print(len(collection))  # Output: 5

  • Method Overloading(Compile-Time Polymorphism)

    While Python does not support method overloading natively (same method name with different arguments), it can be mimicked using default arguments or by handling multiple argument types manually.

    
class Calculator:
    def add(self, a, b=0):
        return a + b

calc = Calculator()
print(calc.add(5))       # Uses default value for b
print(calc.add(5, 3))    # Overridden b

    Techniques for Method Overloading

    Python relies on techniques like dynamic typing and duck typing to achieve similar flexibility and behavior.

    • Dynamic Typing: Dynamic typing means that variables in Python do not have a fixed type. The type of a variable is determined at runtime, based on the value assigned to it. This allows functions, methods, and operators to work with objects of different types, provided the required behavior is supported.
      • 
# A function that works with different types
# Polymorphism with operators
def add(a, b):
    return a + b

# Passing integers
print(add(2, 3))  # Output: 5

# Passing strings
print(add("Hello, ", "World!"))  # Output: Hello, World!

# Passing lists
print(add([1, 2], [3, 4]))  # Output: [1, 2, 3, 4]
    • Duck Typing: The concept of duck typing comes from the phrase: "If it looks like a duck, swims like a duck, and quacks like a duck, then it probably is a duck." In Python, duck typing means that an object's suitability is determined by its behavior (methods and attributes) rather than its explicit type. This allows objects of different types to be used interchangeably, as long as they implement the required behavior.
      • 
class Dog:
    def speak(self):
        return "Bark"

class Cat:
    def speak(self):
        return "Meow"

class Robot:
    def speak(self):
        return "Beep boop"

# A function that works with any object having a 'speak' method
def make_speak(animal):
    print(animal.speak())

# Demonstrating duck typing
dog = Dog()
cat = Cat()
robot = Robot()

make_speak(dog)   # Output: Bark
make_speak(cat)   # Output: Meow
make_speak(robot) # Output: Beep boop

  • Method Overriding (Runtime Polymorphism):

    A subclass can provide a specific implementation of a method that is already defined in its superclass. This allows different behaviors for the same method depending on the object that invokes it.

    
class Animal:
    def sound(self):
        return "Some generic sound"

class Dog(Animal):
    def sound(self):
        return "Bark"

class Cat(Animal):
    def sound(self):
        return "Meow"

# Demonstration
animals = [Dog(), Cat()]
for animal in animals:
    print(animal.sound())
