Rendering Html Templates in python

Rendering HTML templates in Django allows dynamic content to be displayed in a structured format using the render() function. Instead of returning plain text with HttpResponse, Django loads an HTML file from the templates folder and injects dynamic data. In views.py, the render(request, 'template_name.html', context) function passes a dictionary (context) containing variables to the template. The corresponding HTML file uses Djangoâ€™s template syntax ({{ variable }}) to display data dynamically. To make Django recognize the template, the app must be added to INSTALLED_APPS in settings.py, and the template path should be correctly set. This approach enhances flexibility and separates logic from presentation.

Rendering of Dynamic Data using context

In Django, dynamic data is passed from the views.py file to an HTML template using the context dictionary in the render() function. The context contains key-value pairs where keys are variable names accessible in the template, and values are the data being passed.

Views.py


from django.shortcuts import render

def index(request):
    names = ["Abc","def","ghi","jkl","mno"]
    items = {
        "pen": 5,
        "pencil": 10,
        "eraser": 60
    }
    context = {
        "names": names,
        "items": items
    }
    return render(request, 'index.html',context)  # Renders index.html

def contact(request):
    return render(request, 'contact.html')

Django Template Engine

Django's template engine is a system that allows you to embed Python-like logic inside HTML files. It helps dynamically render UI based on conditions, such as checking if a user is logged in or fetching data from a database. With Django templates, we can use if-else conditions, loops (for), filters, and template tags to manipulate and display data.

This code defines a Django template structure with a base template (base.html) and three child templates (index.html, contact.html, and about.html) that extend the base template. The base.html file contains the common structure for all pages, including the section and a {% block start %} where specific content will be inserted by the child templates. Each child template overrides this block to display unique content. The index.html page displays a heading and a list of names and items, applying Django filters and conditions. The contact.html and about.html pages inherit from the base template and provide simple content with navigation links.

Base.html

This file serves as the parent template for all other pages. It contains the basic structure of an HTML document, including , tags, and a section with a {% block start %} placeholder. Any child template that extends base.html will replace the {% block start %} section with its own content while keeping the shared layout intact.

Index.html

This is the home page that extends base.html

  • It includes an h1 tag describing the file location.
  • It displays the variables names and items.
  • It applies Django template filters like: |length to show the length of names,|default to provide a default value if fruits is empty.
  • A {% for %} loop iterates through the names list,It skips the value "def".
  • It includes a contact page link
  <!-- Django Template Code -->
  <{% extends "base.html" %}>
  <{% block start %}>
        <h1>This is index.html file in templates folder which is in home folder</h1>
        {{names}}
        {{items}}
        <!-- To print the length of name -->
        {{names| length}}
        <br>
        <!-- Default value -->
         {{fruits|default:"No Value provided"}}
         {{fruits}}

        <br>
        <{% for name in names %}>
        <!-- Condition to not print the name -->
        <{% if not name == "def" %}>
        <h4>{{name}}</h4>
        <{% endif %}>
        <{% endfor %}>
        <a href="/contact">Contact</a>
  <{% endblock %}>

Conclusion

Reusability in Django templates allows developers to write code once and use it across multiple pages, reducing redundancy and improving maintainability. By leveraging template inheritance ({% extends %}) and reusable components like blocks ({% block %}), templates become modular and easier to manage. This approach enhances scalability, consistency, and efficiency in web development.
