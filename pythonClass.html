Classes

A class in Python is a blueprint or template for creating objects. It defines the structure and behavior of the objects, including attributes (data) and methods (functions). Classes enable object-oriented programming (OOP) principles, such as encapsulation, inheritance, and polymorphism.


class Car:
    def __init__(self, brand, model):
        self.brand = brand
        self.model = model

    def display_info(self):
        print(f"Car Brand: {self.brand}, Model: {self.model}")

Constructor

A constructor is a special method in a class that initializes an object's attributes when the object is created. In Python, the constructor is defined using the __init__ method.

    Features:
  • Automatically called when an object is instantiated.
  • Used to set up initial attribute values.
  • Can take arguments to customize object initialization.

class Person:
    def __init__(self, name, age):
        self.name = name  # Attribute initialization
        self.age = age

# Creating an instance of the Person class
person1 = Person("Alice", 30)
print(person1.name)  # Output: Alice
print(person1.age)   # Output: 30

Instance

An instance is a specific object created from a class. It represents a single, concrete realization of the class blueprint.

    Features:
  • Its own copy of attributes defined by the class.
  • The ability to use the methods defined by the class.

class Animal:
    def __init__(self, species):
        self.species = species

    def speak(self):
        print(f"This is a {self.species}.")

# Creating two instances of the Animal class
dog = Animal("Dog")
cat = Animal("Cat")

dog.speak()  # Output: This is a Dog.
cat.speak()  # Output: This is a Cat.

Use Cases

  • Modeling Real-World Objects: Classes are ideal for representing real-world entities, such as cars, people, or animals, with their attributes and behaviors. 
    
class Student:
    def __init__(self, name, roll_no):
        self.name = name
        self.roll_no = roll_no
  • Encapsulation: Classes can bundle data and methods together while restricting access to internal details. 
    
class BankAccount:
    def __init__(self, balance):
        self.__balance = balance  # Private attribute

    def deposit(self, amount):
        self.__balance += amount

    def get_balance(self):
        return self.__balance
  • Code Reusability: With inheritance, classes allow you to reuse and extend existing functionality. 
    
class Vehicle:
    def __init__(self, brand):
        self.brand = brand

class Car(Vehicle):
    def __init__(self, brand, model):
        super().__init__(brand)
        self.model = model
  • Maintaining State: Classes are great for objects that need to maintain a state across method calls. 
    
class Counter:
    def __init__(self):
        self.count = 0

    def increment(self):
        self.count += 1
        return self.count
  • Custom Data Structures: You can use classes to implement data structures like stacks, queues, linked lists, etc. 
    
class Stack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        return self.items.pop()
  • Working with APIs or Databases: Classes can model and manage the behavior of APIs, database connections, or resources. 
    
class APIClient:
    def __init__(self, api_key):
        self.api_key = api_key

    def fetch_data(self, endpoint):
        # Simulate API call
        print(f"Fetching data from {endpoint} with key {self.api_key}")
  • Building Large Applications: Classes organize code into reusable and modular components, making it easier to manage and scale.
